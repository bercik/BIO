arrays

CREATE_ARRAY (size)

Creates array of size size filled with none elements

Parameters:

size (int) - size of array

Return value:

(array) - array with none elements

Errors:

BAD_PARAMETER_TYPE - if passed parameter is not int

SIZE_LESS_THAN_ZERO - if passed size is less than zero

CREATE_DICT (<key, value>*)

Creates dict with string keys and any values. Initialize it with given key and value (can be repeated)

Parameters:

key (string)

value (all)

Return value:

(dict)

Errors:

BAD_PARAMETER_TYPE - if key is not string

CREATE_TUPLE (<el>*)

Creates tuple of given elements el

Parameters:

el (all) - element to be inserted in tuple

Return value:

(tuple) - tuple of given elements

CREATE_TUPLE_OF_ELEMENTS (size, el)

Creates tuple of size size filled with elements el

Parameters:

size (int) - size of tuple

el (all) - element that will be filled in tuple

Return value:

(tuple) - tuple with given elements

Errors:

BAD_PARAMETER_TYPE - if size isn't int

SIZE_LESS_THAN_ZERO - if size is int but less than zero

SET (a, b, c)

Sets element at given index in array or at given key for dict. If there was value for given key in dict it will be replaced

Parameters:

a (array, dict)

b (int, string) - index for array, key for dict

c (all) - value to be set

Return value:

(all) - value given as parameter c

Errors:

BAD_PARAMETER_TYPE - if a is not array or dict or if b is not int if a is array or string if a is dict

INDEX_OUT_OF_BOUNDS - if given index is greater or equal array size

GET (a, b)

Gets element at given index from array or tuple or at given key from dict

Parameters:

a (array, tuple, dict) - container from which we want to get element

b (int, string) - index for collection, key for dict

Return value:

(all) - returns element from a at given index or key

Errors:

BAD_PARAMETER_TYPE - if a is not array, tuple or dict or or if b is not int if a is collection or string if a is dict

INDEX_OUT_OF_BOUNDS - if given index is greater or equal collection size

KEY_DOESNT_EXIST - if given key doesn't exist in dict

SIZE (a)

Returns size of given collection or dict

Parameters:

a (array, tuple, dict)

Return value:

(int) - size of given collection or dict

Errors:

BAD_PARAMETER_TYPE - If a is not array tuple or dict

RANGE (min, max)

Returns all natural numbers in range of <min, max>

Parameters:

min (int) - lowerbound

max (int) - upperbound

Return value:

(array) - array that contains all natural numbers in given range

Errors:

BAD_PARAMETER_TYPE - if min or max aren't int

MIN_GREATER_THAN_MAX - if min is greater than max

COPY (a)

Copy elements from given array or dict to new one. Note: It won't copy arrays and dicts recursively. To do this use DEEP_COPY instead

Parameters:

a (array, dict) - array or dict to be copy

Return value:

(array, dict) - new array or dict

Errors:

BAD_PARAMETER_TYPE - If a is not array or dict

DEEP_COPY (a)

Copy elements from given array, tuple or dict to new one. Note: It will copy arrays, tuples and dicts recursively. If you don't want this behavior use COPY instead

Parameters:

a (array, tuple, dict) - array, tuple or dict to be copy

Return value:

(array, tuple, dict) - new array, tuple or dict

Errors:

BAD_PARAMETER_TYPE - If a is not array, tuple or dict

SLICE (col, start, end)

Returns new array or tuple with copied objects from given col range <start, end)

Parameters:

col (array, tuple) - collection to be sliced

start (int) - start index of slice

end (int) - end index of slice

Return value:

(array, tuple) - Returns new array if you pass array or new tuple if you pass tuple

Errors:

BAD_PARAMETER_TYPE - If col is not array or tuple or if start or end are not int

START_GREATER_THAN_END - If start is greater than end

INDEX_OUT_OF_BOUNDS - If start or end are out of col bounds

EXTEND_ARRAY (arr, size)

Extends array to given size or if size is less than actual cut it. Fills elements above size with nones. Note: it will return extended array as new array, not affecting given one

Parameters:

arr (array) - array to be extended

size (int) - new size

Return value:

(array) - new extended array

Errors:

BAD_PARAMETER_TYPE - If arr is not array or size is not int

SIZE_LESS_THAN_ZERO - If size is less than zero

UNPACK (id, <id>*, col)

Unpacks collection to given variables. If size of collection is greater than number of variables it will assign all unassigned values as a tuple to the last variable. If number of variables is greater than size of collection than error would be returned. If error occurs during execution of this function than no variable would be affected by it

Parameters:

id (id)

col (array, tuple)

Return value:

(array, tuple) - given collection

Errors:

BAD_PARAMETER_TYPE - If col is not array or tuple

NUMBER_OF_VARIABLES_GREATER_THAN_COLLECTION_SIZE - If number of variables is greater than collection size

CONTAINS (col, el)

Checks whether collection col contains element el. Uses EQ function to achive that

Parameters:

col (array, tuple)

el (all except error)

Return value:

(bool) - true if collection contains element, otherwise false

Errors:

BAD_PARAMETER_TYPE - If one of element in col is error or el is error

Parameters:

Removes key from dict

COUNT_ELEMENTS (a, el)

Count occurences of element el in collection or dict a. Uses EQ function to achive that

Parameters:

a (array, tuple, dict)

el (all except error)

Return value:

(int) - number of occurence of element el in collection or dict a

Errors:

BAD_PARAMETER_TYPE - If a is not collection or dict or one of their element is error or el is error

SORT_ASC (col)

Sorts given collection in ascending order. Collection have to have only strings or numbers to be sorted. Strings are sorted alphabetically. In case of numbers their types remains the same. Returns sorted collection as new array

Parameters:

col (array, tuple) - collection to be sorted

Return value:

(array) - sorted collection as new array

Errors:

BAD_PARAMETER_TYPE - if col is not collection, contains other elements than strings or numbers or there are mixed strings and numbers

SORT_DESC (col)

Sorts given collection in descending order. Collection have to have only strings or numbers to be sorted. Strings are sorted alphabetically. In case of numbers their types remains the same. Returns sorted collection as new array

Parameters:

col (array, tuple) - collection to be sorted

Return value:

(array) - sorted collection as new array

Errors:

BAD_PARAMETER_TYPE - if col is not collection, contains other elements than strings or numbers or there are mixed strings and numbers
