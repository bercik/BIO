Data types

array - series of elements of any type. Mutable

bool - has only two posible values: true and false. Immutable

dict - collections of pairs <string, any>. Mutable

error - contains informations about error that appears. Immutable

float - single-precision 32-bit IEEE 754 floating point. Immutable

int - 32-bit signed two's complement integer, which has a minimum value of -231 and a maximum value of 231-1. Immutable

none - represents absent of value. Immutable

string - represents sequence of characters. Immutable

tuple - series of elements of any type. Immutable

observer

ATTACH_TO_EVENT (event, callback, <event, callback>)

Attach callback functions to event functions. When event function is called, then callback functions attached to it are executed in the same order that they were added and at the end event function is executed. If there are attached callbacks to this callbacks there are also executed in the same manner. Values returned form callback functions are ommited. If error occurs during execution of this function then no callback would be attached to event

Parameters:

event (id) - function to be observed

callback (id) - function to be attached

Return value:

(none)

Errors:

CALLBACK_ALREADY_ATTACHED - if callback is already attached to event

IS_ATTACHED (event, callback)

Checks whether callback is attached to event

Parameters:

event (id)

callback (id)

Return value:

(bool) - true if callback is attached to event, otherwise false

DETACH_FROM_EVENT (event, callback, <event, callback>)

Detach callback functions from event functions. If error occurs during execution of this function then no callback would be detached from event

Parameters:

event (id) - observed function

callback (id) - function to be detached

Return value:

(none)

Errors:

CALLBACK_NOT_ATTACHED - if callback is not attached to event

math

SUM (col)

In case of numbers returns sum of numbers in collection col. In case of strings returns concatenation of strings in collection col. If size of collection col is zero then none would be returned.

Parameters:

col (array, tuple) - collection to be sumed

Return value:

(int, float, string) - If you passed collection of strings then return value would be string. If you passed collection of ints then return value would be int. If you passed collection of floats and ints then return value would be float. If you passed collection of size zero, then none would be returned

Errors:

BAD_PARAMETER_TYPE - if col isn't array or tuple or if collection col contains other data then int, float, string or there are mixed strings and numbers in collection

ADD (val, val, <val>)

In case of numbers return sum. In case of strings return concatenation. In case of collections return tuple with sum of elements at index. Size of given collections must be the same. Internally uses SUM function to achieve this.

Parameters:

val (int, float, string, array, tuple) - value to be sumed

Return value:

(int, float, string, tuple) - If you passed strings then return value would be string. If you passed ints then return value would be int. If you passed floats and ints then return value would be float. If you passed array or tuple then return value would be tuple

Errors:

BAD_PARAMETER_TYPE - if val isn't int, float, string, array or tuple or there are mixed strings, numbers, arrays and tuples or in arrays or tuples the elements at the same index are of bad type or mixed (as explained in SUM function)

COLLECTIONS_DIFFRENT_SIZES - if given collections sizes differs
