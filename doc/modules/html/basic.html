ASSIGN_LOCAL (id, val, <id, val>*)

Creates or updates local variables with identifier id and values val. Caution be carefull when you mix variable assigning and getting their value. For example if you do: ASSIGN_LOCAL(i, 0, j, i) you will get an error about non existing local variable i. You must split above instruction in two.

Parameters:

id (id) - variable identifier

val (all) - variable value

Return value:

(none)

ASSIGN_GLOBAL (id, val, <id, val>*)

Creates or updates global variables with identifier id and values val. Caution be carefull when you mix variable assigning and getting their value. For example if you do: ASSIGN_GLOBAL(i, 0, j, GET_GLOBAL(i)) you will get an error about non existing global variable i. You must split above instruction in two.

Parameters:

id (id) - variable identifier

val (all) - variable value

Return value:

(none)

IS_LOCAL (id)

Checks whether variable with identifier id exists and is local

Parameters:

id (id) - variable identifier

Return value:

(bool) - true if variable exists, otherwise false

GET_LOCAL (id)

Returnes local variable with identifier id. Instead of using this function you can just type identifier in place where you want to get local variable

Parameters:

id (id) - local variable identifier

Return value:

(all) - local variable value

Errors:

NO_LOCAL_VARIABLE - if there is no local variable with id identifier

IS_GLOBAL (id)

Checks whether variable with identifier id exists and is global

Parameters:

id (id) - variable identifier

Return value:

(bool) - true if variable exists, otherwise false

GET_GLOBAL (id)

Returnes global variable with identifier id

Parameters:

id (id) - global variable identifier

Return value:

(all) - global variable value

Errors:

NO_GLOBAL_VARIABLE - if there is no global variable with id identifier

RETURN (val)

Returnes value val from function

Parameters:

val (all) - value to be returned

EXIT ()

Exits from BIO

onUNHANDLED_ERROR (err)

Event called when there is error value returned from onSTART function or unhandled error during program execution (unhandled that is not assigned to variable or passed to function) Note: if you don't assign any observer to this event than err would be printed on screen and program exited

Parameters:

err (error) - error returned from onSTART function or unhandled
