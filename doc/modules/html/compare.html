compare

LS (a, b)

Compare if number a is less then number b or if numbers at certain index in collection a are less then numbers at the same index in collection b

Parameters:

a (int, float, tuple, array)

b (int, float, tuple, array)

Return value:

(bool, array) - if you pass two numbers then the result will be bool. However if you pass two collections the result will be array of bools, where every element in tuple corresponds to result of operation LS on numbers at this index in given collections

Errors:

BAD_PARAMETER_TYPE - if a or b aren't int, float, tuple or array or if you mix numbers with collections or if elements in collections aren't numbers

COLLECTIONS_DIFFRENT_SIZES - if you pass collections and their sizes are diffrent

EQ (a, b)

Compare if element a is equals element b. Two elements must be the same type to be considered equals (except numbers (ints and floats) and collections (arrays and tuples)). Two elements are considered to be equals in various cases:
- Underlying references are the same
- Numbers, strings, bools have got the same value
- Nones are always equals
- Dicts have got the same keys and values at these keys are equals
- Collections have got the same sizes and values at the same index are equals
Note: you can't compare two errors. Instead compare one of its element like object or message

Parameters:

a (all except error)

b (all except error)

Return value:

(bool) - true if passed values are equals, otherwise false

Errors:

BAD_PARAMETER_TYPE - If at least one passed value is error

NEQ (a, b)

Compare if element a isn't equals element b. It uses EQ function internally and just negate it result. Check EQ function for details

Parameters:

a (all)

b (all)

Return value:

(bool) - true if passed values aren't equals, otherwise false

IS_IN_RANGE (num, min, max)

Checks if number num is in range of <min, max>

Parameters:

num (int, float)

min (int, float)

max (int, float)

Return value:

(bool) - true if num is in range, otherwise false

Errors:

BAD_PARAMETER_TYPE - if any of parameter are not number

MIN_GREATER_THAN_MAX - if min is greater than max
